Task 4

Barycentric coordinates are a convenient way to describe a point within a triangle. Each value in a barycentric vector corresponds to an offset that the point has from a vertex of the triangle. If all values are between 0 and 1 then the point is within the triangle, if any of the points are negative or greater than 1, then the point is outside the defined triangle.

Task 5

Pixel sampling involves taking our sample point in our picture and finding its corresponding barycentric coordinates. Then we use the coordinates in the texture to create a matrix which is used to convert our barycentric coordinates to (u, v) coordinates. These are the values we use to then sample our data. The two sampling methods we use are: Nearest and Bilinear. The nearest sampling method involves using the pixel in the texture that is the closest to our (u, v) point. We can do this by simply rounding our vector. The bilinear method finds the 4 closest pixels around our (u, v) point and lerps the colors of those pixels, based on how far they are from our original point. The final resulting color is used.

The biggest differences between the images is that the locations where there are high frequencies such as the while grid lines, with the nearest sampling method, the lines are sometimes missed. By using bilinear we are able to faintly bring those high frequencies back.

Task 6

Texturing is the process of mapping the location in screen space to a location in texture space. By calculating the rate of change between pixel locations in screen space vs texture space, we can determine if an image should be stretched or flat. Level sampling allows us to handle the scenarios where a screen pixel is either much bigger or much smaller than a pixel in texture pixel. A texture is often saved in different ‘levels’ which correspond to how big the texture is. When a screen pixel is too small, we want to use lower levels in order to get access to more finer details. When a pixel is big, we use higher levels so we can capture the important parts of the image.

To implement level sampling, you first must calculate the proper level to sample from. This is done by using the barycentric points (x,y)(x,y)(x,y)﻿ (x+1,y)(x+1,y)(x+1,y)﻿ (x,y+1)(x,y+1)(x,y+1)﻿ and using the difference between these points as differentials. After scaling these differentials with our screen size we can calculate the correct level as a float. If we are using a nearest sampling method, then we just round the level to the nearest integer and use its corresponding mipmap. If we are using bilinear, then we the floor and ceiling of our float and take the average after sampling both values.

We can now compare the tradeoffs of our different sampling techniques

  • Samples per Pixel: Very good antialiasing power but uses a lot more memory and is rather slow
  • Pixel Sampling: Good antialiasing, good memory storage and fast
  • Level Sampling: Good antialiasing and good memory storage but slow
